## 🛠️ Step 1: Core Technologies & Libraries

To handle real-time transcription and a floating UI on a PC, these are your best bets:

* **Transcription Engine:** Use `Faster-Whisper`. The standard OpenAI Whisper is slow for real-time; `Faster-Whisper` is a reimplementation using CTranslate2 that is up to 4x faster and uses less memory.
* **Audio Handling:** `PyAudio` or `SoundCard` for capturing real-time microphone input, and `librosa` or `pydub` for processing imported files.
* **GUI Framework:** `CustomTkinter` or `PyQt6`. Both allow for "frameless" and "always-on-top" windows, which are essential for a floating widget.
* **Packaging:** `Nuitka` or `PyInstaller`. Nuitka is generally better for AI projects as it compiles Python to C, often resulting in faster execution and smaller file sizes.

## 📋 Step 2: The Development Plan

### Phase 1: Real-Time Transcription Logic

1. **Audio Chunking:** You can't feed a continuous stream to Whisper directly. You must capture audio in small "chunks" (e.g., 2–3 seconds) or use **Voice Activity Detection (VAD)** to only send audio when someone is speaking.
2. **Threading:** Run the Whisper model in a background thread. If you run it in the same thread as your GUI, the widget will freeze every time it processes a sentence.

### Phase 2: Building the Floating Widget

1. **UI Design:** Create a small, minimalist window.
2. **Windows Specifics:** Use flags like `attributes('-topmost', True)` in Tkinter to keep it above all other apps.

### Phase 3: File Import Feature

1. **Drag and Drop:** Implement a simple "Upload" button or drag-and-drop functionality using `Tkinterdnd2`.
